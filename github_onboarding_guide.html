STEP 01

Git & VS Code 설치

개발에 필요한 두 가지 프로그램을 설치합니다

🔧
필수 설치
Git 설치하기
▼

Git은 코드의 변경 이력을 관리하고 GitHub와 연결해주는 핵심 도구입니다. 설치하지 않으면 아무것도 할 수 없어요.

📥
Git for Windows 다운로드
git-scm.com — 공식 사이트에서 설치 파일 다운로드
↗
  • 1위 링크를 클릭하면 자동으로 설치 파일(.exe) 다운로드가 시작됩니다
  • 2다운로드된 파일을 더블클릭해서 실행합니다
  • 3설치 창이 열리면 모든 항목에서 그냥 Next 를 눌러 기본값 그대로 설치합니다
  • 4설치 완료 후 "Git Bash Here" 체크박스가 있으면 켜둡니다 (기본값으로 켜져 있음)
💡 설치가 완료되면 바탕화면이나 폴더에서 마우스 오른쪽 버튼을 눌렀을 때 "Open Git Bash here" 메뉴가 생깁니다. 이게 보이면 성공입니다.
📝
필수 설치
VS Code 설치하기
▼

VS Code(Visual Studio Code)는 코드를 수정하는 편집기입니다. 메모장보다 훨씬 편리하고 무료입니다.

💻
VS Code 다운로드
code.visualstudio.com — Download for Windows 버튼 클릭
↗
  • 1파란색 "Download for Windows" 버튼을 클릭합니다
  • 2다운로드된 .exe 파일을 실행합니다
  • 3설치 중 "PATH에 추가" 옵션이 있으면 반드시 체크합니다
  • 4나머지는 기본값 그대로 Next → Install → Finish
✅ 설치 완료 후 VS Code를 열면 왼쪽에 파일 탐색기, 가운데에 코드 편집 창이 보입니다. 한국어 팩을 설치할지 물어보면 설치해도 됩니다.
⚙️
초기 설정
Git에 내 이름 등록하기
▼

Git에 커밋(수정 저장)할 때 "누가 수정했는지" 표시됩니다. 처음 한 번만 설정하면 됩니다.

바탕화면 빈 곳에서 마우스 오른쪽 버튼 → "Open Git Bash here" 를 클릭해서 Git Bash를 엽니다. 그 다음 아래 명령어를 입력합니다.

Git Bash 터미널
# 이름 설정 (따옴표 안에 본인 이름을 입력하세요) git config --global user.name "홍길동" # 이메일 설정 (GitHub 가입할 이메일 주소) git config --global user.email "your@email.com"
⚠️ 이름과 이메일은 다음 단계에서 만들 GitHub 계정과 같은 이메일을 사용하는 것이 좋습니다.
STEP 02

GitHub 계정 만들기

코드를 올리고 공유하는 플랫폼 계정을 생성합니다

🐱
계정 생성
GitHub 회원가입
▼
🐙
GitHub 회원가입 페이지
github.com/signup
↗
  • 1Email: 자주 쓰는 이메일 입력
  • 2Password: 영문+숫자 조합 8자 이상
  • 3Username: 영문으로 본인이 원하는 닉네임 (나중에 바꿀 수 있어요)
  • 4이메일 인증 코드를 입력하면 가입 완료
  • 5가입 후 계정 아이디(Username)를 담당자(시형 님)에게 알려주세요 — 레포 초대를 보내드립니다
💡 GitHub는 전 세계 개발자들이 코드를 올리고 공유하는 플랫폼입니다. 무료로 사용할 수 있어요.
STEP 03

초대 수락 & 코드 가져오기

레포지터리에 참여하고 내 PC로 코드를 복사합니다

📨
초대 수락
이메일 초대 수락하기
▼
  • 1시형 님이 GitHub에서 Collaborator로 초대를 보내드립니다
  • 2가입한 이메일로 "You've been invited to collaborate" 메일이 도착합니다
  • 3메일 안의 "Accept invitation" 버튼을 클릭합니다
  • 4GitHub 사이트로 이동되며 수락 확인 버튼을 한 번 더 누릅니다
⚠️ 초대 이메일이 스팸함에 들어가 있는 경우도 있으니 확인해보세요. 7일 내에 수락하지 않으면 초대가 만료됩니다.
📦
코드 내려받기
Clone — 내 PC로 가져오기
▼

Clone이란 GitHub 서버에 있는 코드를 내 컴퓨터에 복사해 오는 것입니다. 처음 한 번만 하면 됩니다.

  • 1내 컴퓨터의 문서(Documents) 폴더 또는 원하는 폴더를 탐색기로 엽니다
  • 2폴더 빈 곳에서 마우스 오른쪽 버튼 → "Open Git Bash here" 클릭
  • 3검은 창(Git Bash)이 열리면 아래 명령어를 입력합니다
Git Bash 터미널
git clone https://github.com/curricenterhscne/transfer_cne.git
  • 4GitHub 로그인 창이 팝업되면 본인 계정으로 로그인합니다
  • 5다운로드가 완료되면 현재 폴더 안에 transfer_cne 폴더가 생깁니다
  • 6이 폴더가 이제 내 작업 공간입니다!
✅ Clone이 성공하면 Cloning into 'transfer_cne'... 메시지가 보이고, 폴더가 만들어집니다.
🗂️
VS Code로 열기
폴더를 VS Code에서 열기
▼
  • 1VS Code를 실행합니다
  • 2메뉴에서 파일(File) → 폴더 열기(Open Folder) 클릭
  • 3방금 생성된 transfer_cne 폴더를 선택합니다
  • 4왼쪽 패널에 파일 목록이 나타나면 성공입니다
💡 또는 Git Bash에서 cd transfer_cne && code . 를 입력하면 바로 VS Code가 열립니다.
STEP 04

파일 수정하기

VS Code에서 코드를 편집합니다

✏️
기본 편집
VS Code에서 HTML 파일 수정하기
▼
  • 1왼쪽 파일 탐색기에서 수정할 파일(예: index.html)을 클릭합니다
  • 2가운데 편집 창에 내용이 표시됩니다. 원하는 부분을 클릭해서 수정합니다
  • 3수정 후 Ctrl+S로 저장합니다
  • 4파일명 옆에 ● 점이 생기면 저장 안 된 상태 / 사라지면 저장 완료
💡 VS Code에서 HTML 파일을 열면 태그가 색깔별로 구분되어(Syntax Highlighting) 훨씬 읽기 편합니다. Ctrl+Z로 실수를 되돌릴 수도 있어요.
⚠️ 파일을 수정하기 전에 반드시 최신 코드를 먼저 받아야(pull) 합니다. 5단계에서 설명합니다.
STEP 05

수정 내용 올리기

내가 수정한 코드를 GitHub에 저장합니다 — commit & push

🔄
먼저 항상
작업 전: 최신 코드 받기 (pull)
▼

작업을 시작하기 전에 항상 pull을 먼저 해서 다른 팀원이 올린 최신 코드를 받아옵니다. 이걸 빠뜨리면 충돌이 생길 수 있어요.

transfer_cne 폴더 안에서 Git Bash를 열고:

Git Bash 터미널
# transfer_cne 폴더 안으로 이동 (처음 열 때만) cd transfer_cne # 최신 코드 받기 git pull origin main
✅ 이미 최신 상태라면 "Already up to date." 라고 표시됩니다.
📸
변경 저장
commit — 수정 내용 기록하기
▼

commit은 수정한 내용을 "이 시점에 이렇게 저장했다"고 기록하는 행위입니다. 사진을 찍는 것과 비슷해요.

Git Bash 터미널
# 변경된 모든 파일을 스테이징 (올릴 파일 선택) git add . # 커밋 메시지 작성 (무엇을 수정했는지 간단히) git commit -m "전입학 절차 안내 내용 수정"
💡 커밋 메시지는 한국어로 써도 됩니다. "무엇을 왜 수정했는지" 짧게 적으면 나중에 이력 확인이 편합니다. 예: "학교 목록 3개 추가", "지도 필터 버그 수정"
🚀
서버에 올리기
push — GitHub에 업로드
▼

push는 내 컴퓨터에 저장한 commit을 GitHub 서버에 올리는 것입니다. push해야 다른 팀원도 내 수정 내용을 볼 수 있어요.

Git Bash 터미널
git push origin main
  • 1처음 push할 때 GitHub 로그인 팝업이 열릴 수 있습니다. 로그인하면 됩니다
  • 2완료 후 GitHub 사이트에서 수정 내용이 올라가 있는지 확인해보세요
✅ push가 성공하면 https://github.com/curricenterhscne/transfer_cne 에서 내가 수정한 내용을 확인할 수 있습니다.
STEP 06

충돌 해결하기

같은 파일을 동시에 수정했을 때 발생하는 충돌을 해결합니다

⚡
충돌이란?
Conflict가 발생하는 상황
▼

두 사람이 같은 파일의 같은 줄을 동시에 다르게 수정하면 Git이 어느 쪽을 선택해야 할지 몰라서 충돌(Conflict)이 발생합니다.

충돌이 발생한 파일 예시
<<<<<<< HEAD (내가 수정한 내용) <h1>충청남도 고등학교 전입학 안내</h1> ======= <h1>충남 고등학교 전학 안내</h1> >>>>>>> origin/main (상대방이 올린 내용)
  • 1VS Code에서 해당 파일을 열면 충돌 부분이 색깔로 표시됩니다
  • 2"Accept Current Change"(내 것 선택) 또는 "Accept Incoming Change"(상대방 것 선택) 버튼을 클릭합니다
  • 3또는 두 내용을 함께 합칠 수도 있어요
  • 4수정 후 저장 → 다시 git add . → git commit → git push
⚠️ 충돌이 어렵게 느껴지면 담당자(시형 님)에게 연락해 주세요. 함께 해결하면 됩니다. 충돌이 발생했다고 데이터가 삭제되지는 않습니다.
🛡️
충돌 예방
충돌을 줄이는 습관
▼
  • 1작업 전 pull 먼저 — 항상 최신 코드를 받고 수정 시작
  • 2담당 파일 분리 — 서로 다른 파일을 맡아서 수정 (예: 시형 님은 map.js, 팀원은 survey.js)
  • 3수정 전 채팅으로 공유 — "지금 index.html 수정할게요" 미리 알리기
  • 4자주 commit & push — 오래 들고 있을수록 충돌 위험 증가
STEP 07

일상적인 작업 흐름 정리

매일 반복하는 기본 루틴입니다

📋
전체 루틴
하루 작업 순서 요약
▼
매일 작업하는 순서 (Git Bash)
# 1. 프로젝트 폴더로 이동 cd ~/Documents/transfer_cne # 2. 최신 코드 받기 (항상 먼저!) git pull origin main # --- VS Code에서 파일 수정 후 --- # 3. 변경 파일 스테이징 git add . # 4. 커밋 (무엇을 수정했는지 메시지 작성) git commit -m "수정 내용을 간단히 적기" # 5. GitHub에 업로드 git push origin main
🎉 이 5단계만 반복하면 됩니다. 처음에는 낯설지만 2~3번 해보면 금방 익숙해집니다. 모르는 것이 생기면 언제든 시형 님께 연락 주세요!
